Here are the data structures with brief descriptions:
 CAngleToPlayerThis script calculates the angle between this object and the player, and flips the sprite depending on where the player is. It's useful for 2D enemies or NPCs that need to "look" at the player
 CDoorInteractableControls door interaction behavior (opening/closing) using animations. Implements the IInteractable interface for player interactions
 CEnemyControllerControls the enemy's AI: patrol, chase, and attack logic. Designed for a survival horror style game
 CEnemySpriteLookMakes the enemy sprite always face the player. Optionally, the enemy can also look up/down if needed. Super useful for 2.5D or top-down games where you want the enemy to "track" the player visually
 CGameManagerHandles core game management actions such as scene transitions and quitting the game. Simple and reusable for menus, checkpoints, or any global game logic
 CHolderInteractableAllows the player to place or pick up objects on a holder (like a shelf or pedestal). Provides UI feedback and visual cues for correct placement
 CIInteractableInterface for any object that can be interacted with by the player. The optional parameter allows passing the object currently held by the player (if any)
 CInventoryManagerManages the player's inventory, including UI updates, item storage, and equipment logic. Designed for a survival horror game where the player can pick up, equip, and combine items
 CItemDefines inventory items as ScriptableObjects for easy asset creation and management. Used for both 2D UI representations and 3D in-game objects
 CItemControllerHandles item pickup, inventory integration, and player interaction logic. Supports both equippable items (like the phone) and consumables
 CMouseLookHandles first-person mouse look for both the player body and the camera. Disables look controls when the inventory is open
 CPlayerControllerControls core player functionality including movement, interactions, and inventory management. Designed for a 3D survival horror experience with physics-based movement
 CPS2InteractableHandles the PS2 console interaction: accepts a specific DVD, plays video, and updates UI. Designed for a puzzle or Easter egg in a survival horror game
 CStatsManages player stats (health and sanity), updates the UI, and triggers world corruption effects. Designed for a survival horror game where the world becomes more unstable as sanity decreases
 CUITextControllerControls the display, style, and animation of in-game UI messages. Supports different message types (interact, collect, collected, read) with typing and fade effects
